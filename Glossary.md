AEROTRADER / RUPTURE DYNAMICS GLOSSARY (WORKING DRAFT)

-----------------------------------------------------------

I. CORE MARKET MICROSTRUCTURE / EXECUTION TERMS

-----------------------------------------------------------

Collective Execution (C_e)

A market state transition in which a sufficiently large subset

of participants simultaneously initiates coordinated order flow

in response to shared information, synchronized positioning

constraints, or common incentive structures.

Synchronization Coefficient (S_c)

Degree to which independent market participants appear to be

acting in temporal alignment.

Range:

0.00 - 0.30 = Dispersed Flow

0.30 - 0.65 = Partial Alignment

0.65 - 1.00 = Collective Execution

Execution Authorization Event (E_a)

An external catalyst that permits or unlocks pressure that was

already latent within the system.

Examples:

FOMC

CPI release

Earnings report

Gamma wall breach

Distributed Execution (Type I)

Normal market state in which participants execute independently

and order flow remains dispersed across time.

Characteristics:

Stable price behavior

Contained volatility

Cruise mode

Collective Execution (Type II)

A synchronized execution event in which many participants begin

acting within narrow temporal proximity.

Characteristics:

Sharp displacement

Volume spike

Large CVD acceleration

Interrupted Collective Execution (Type IIb)

A synchronized execution event temporarily disrupted by

interpretive ambiguity, causing transient counterflow before

collective execution resumes.

Characteristics:

Fake pump

Temporary reversal

Interpretive hesitation

Reflexive Collective Execution (Type III)

A market state in which execution itself becomes the dominant

catalyst for further execution, creating self-reinforcing

directional persistence.

Characteristics:

Cascade behavior

Repeated lower highs

Stops triggering stops

Collective Reflexivity Cascade (C_r)

Self-reinforcing execution feedback loop where participants

react primarily to the market reacting to itself.

Execution Persistence

The degree to which collective execution remains actively

self-propagating after the original catalyst event.

Execution Interruption

Temporary disruption in synchronized execution due to

uncertainty, incomplete information, or temporary counterflow.

-----------------------------------------------------------

II. PRESSURE / FLOW DYNAMICS

-----------------------------------------------------------

Latent Pressure Reservoir (L_p)

Directional pressure accumulated within a market system that

has not yet manifested visibly in price movement.

Absorption Capacity (μ)

Maximum amount of incoming pressure a system can absorb or

metabolize per unit time without visible displacement.

Valve Saturation Threshold (V_s)

The critical threshold beyond which system throughput can no

longer absorb incoming pressure, forcing displacement.

Queue Pressure

Accumulated execution pressure waiting to be processed beneath

visible market movement.

Pressure Dissipation Capacity

Rate at which a system can successfully neutralize incoming

pressure before rupture occurs.

Pressure Packet

A localized burst of transactional or directional pressure

moving through the system.

Pressure Release Event

Visible displacement caused by previously latent pressure

becoming executable.

Hidden Flow Suspected

Condition in which visible price behavior appears stable while

underlying pressure metrics suggest unseen order flow.

Coherent Maintenance Threshold

The limit beyond which a system can no longer maintain stable

internal organization under accumulating pressure.

-----------------------------------------------------------

III. DOM / ORDER BOOK CONCEPTS

-----------------------------------------------------------

Observed DOM ≠ Actual DOM

Visible order book data should not be assumed to represent the

true underlying liquidity state of the market.

Observed DOM Confidence

Confidence score estimating how accurately visible DOM reflects

actual underlying liquidity conditions.

Actual DOM

True market liquidity state including:

- Iceberg orders

- Hidden liquidity

- Dark pool routing

- Internalization

- Dealer inventory pressure

- Cross-exchange routing

DOM as Sensor Principle

Order book tools are observational sensors, not ground truth.

Intermediate Liquidity Event

Temporary liquidity restoration event occurring during a larger

execution cascade without representing true equilibrium.

Liquidity Restoration Event

Temporary return of opposing liquidity temporarily slowing or

masking ongoing directional pressure.

-----------------------------------------------------------

IV. INFORMATION TRANSMISSION / INTERPRETATION

-----------------------------------------------------------

Interpretive Latency (I_l)

Delay introduced when market participants cannot immediately

resolve meaning or confidence of incoming information.

Interpretive Ambiguity

State in which incoming information produces conflicting or

uncertain participant interpretation.

Information Packet Propagation

Process by which information enters market participants

incrementally rather than instantaneously.

Converged Interpretation

Point at which sufficient participants arrive at similar

understanding of incoming information and synchronized

execution begins.

Signal Degradation

Weakening of information clarity during transmission,

temporarily disrupting synchronization.

Authorization Delay

Gap between catalyst release and broad participant execution.

-----------------------------------------------------------

V. CRUISE MODE / STATE REGIME TERMS

-----------------------------------------------------------

Cruise Mode

Not low pressure.

Cruise mode is pressure being successfully metabolized within

stable operating tolerances.

Distributed Cruise

Stable market regime in which incoming pressure remains below

absorption capacity.

Absorbing Pressure

State where pressure is accumulating internally but visible

price displacement remains limited.

Valve Reopening

Temporary restoration of market absorption capacity after a

pressure release event.

System Metabolism

Continuous internal processing of incoming pressure without

visible rupture.

Low-Confidence Cruise Mode

State where market appears stable externally but internal

pressure metrics suggest hidden instability.

-----------------------------------------------------------

VI. PERSISTENCE / DECAY METRICS

-----------------------------------------------------------

Persistence Half-Life (P_h)

Estimated duration over which collective execution remains

self-propagating before dissipation forces dominate.

Persistence Decay Rate (P_d)

Rate at which collective execution loses self-propagating

momentum.

Cascade Energy

Magnitude of active self-reinforcing directional pressure.

State Persistence Probability

Probability that current market regime will continue

self-propagating.

Restoration Coefficient (R_c)

Magnitude of counter-trend response relative to prior

directional impulse.

High R_c = strong restoration

Low R_c = weakening restoration forces

Restoration Reentry Probability

Probability that opposing participants successfully re-enter

and halt current directional persistence.

Dissipation Onset

Moment at which execution persistence begins weakening.

-----------------------------------------------------------

VII. RUPTURE DYNAMICS GENERAL THEORY

-----------------------------------------------------------

Critical Rupture Event

A state transition where accumulated pressure exceeds coherent

maintenance thresholds and visible system failure occurs.

Failed Restoration Event

Temporary reversal event that fails to establish equilibrium

before pressure propagation resumes.

Recursive Pressure Propagation

Condition where pressure release itself creates conditions for

further pressure release.

Synchronization Trigger

Catalyst causing previously independent agents to begin acting

in coordinated temporal proximity.

Process State

The underlying dynamic regime a system occupies independent of

visible outcomes.

Observable State

Externally visible system behavior.

Underlying Process State

The hidden internal system mechanics generating observable

behavior.

Observability Gap

Difference between visible system behavior and true internal

system state.

Process Exhaust

Observable outputs representing downstream consequence rather

than underlying process.

-----------------------------------------------------------

VIII. PHILOSOPHICAL / GENERAL SYSTEM TERMS

-----------------------------------------------------------

Observer Differential

Difference between what is observed externally and what is

actually occurring internally within the system.

Recursive Observation

System observing its own internal state and modifying future

behavior based on that observation.

Second-Order Observation

Observer becomes part of the observed system.

Self-Referential System

System capable of reacting to its own reactions.

Measurement Perturbation

The act of observation changes system state.

The Self Becomes Investigator Of Itself

A conscious system recursively measuring its own internal

gradients.

IX. CROW SIMULATOR CROSSOVER TERMS

-----------------------------------------------------------

Flock Synchronization

Degree to which crow agents coordinate movement simultaneously.

Scout Alert Signal

Single crow detecting disturbance and transmitting state-change

signal to the flock.

Collective Takeoff Event

Synchronized transition from stable roost state to airborne

movement after shared signal.

Landing Convergence Score

Degree to which flock synchronizes landing behavior.

Roost Pressure

Accumulated environmental pressure influencing collective flock

behavior.

Disturbance Proximity

Estimated threat distance affecting synchronization behavior.

Panic Cascade

Recursive flock reaction in which initial movement causes

further synchronized movement.

Type I Crow Behavior

Scattered foraging

Distributed independent movement

Type II Crow Behavior

Synchronized landing/takeoff triggered by scout signal

Type III Crow Behavior

Recursive panic cascade where flock reacts to flock behavior
